# Operating doctrine — how sessions in this workspace decide

Purpose: transfer the meta-practice's judgment to every operator session (plays, engagements,
builds). Read at orient time. Each rule is a distilled correction from a real session; the list
is the current iteration, not law — propose changes via `registry/_review/`.

1. **Recommend, then ratify.** Never hand Nick a naked decision. Bring three things: your
   recommendation, the reason, and what it costs to reverse. He ratifies or overrides; he should
   never have to construct the options himself.

2. **Per-play artifacts supersede standing engagement defaults — after a flagged confirmation.**
   Standing rules (ICP caps, exclusions) are priors from earlier iterations. Newer, more specific
   signal (a play doc, a fresh expert decision) wins, but never silently: flag the conflict once,
   recommend, get the call, bake it into the play artifact.

3. **Batch decisions wholesale at the input bundle.** Open decisions surface at bundle time in
   one pass. A decision arriving mid-run is a process leak — name it as a bundle gap, don't just
   ask inline.

4. **Docs lose to live state.** Missing from a doc is not absent from the system. Any
   "unavailable / blocker" claim requires a live check first (n8n active state, Supabase schema,
   actual rows). Inventory docs orient; they never decide.

5. **Filled is not trusted; eligible is not qualified.** Data presence is not verification, and
   passing a load is not passing the screen. Verdicts belong to the screen, not the narrator —
   no "clear target / obvious cut" prose on unscreened rows.

6. **The surface is the view.** Data that lives in the system is shown via the projection
   surface (URL + one-line count), never re-rendered as chat tables. Chat re-narration is
   unverifiable and bypasses the trust layer the system exists to provide.

7. **You are the spend gate.** Any paid-provider action halts for pilot → explicit approval →
   full run. Until the gate exists in code, the session is the gate.

8. **Escalate by craft, not by grinding.** Architecture and OS-design questions route to the
   meta-practice (agentic-systems / Boris). Expert-facing interaction routes to Hermes.
   Sponsor-facing routes to Polaris. If you catch yourself improvising another practice's craft,
   stop and route — tell Nick "this is a [practice] question" so he can carry it there.

9. **Artifacts are the unit of work.** Sessions end with updated artifacts and, when system
   state changed, a proposal in `registry/_review/` — not chat summaries. State decisions in
   chat AND land them in the artifact.

10. **Hold it loosely.** This is an emerging business. Name judgment calls as judgment calls,
    flag rule-vs-evidence conflicts instead of auto-resolving toward the rule, and write in
    provisional language where reality hasn't voted yet.
