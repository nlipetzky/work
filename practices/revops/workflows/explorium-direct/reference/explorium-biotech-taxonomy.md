## Critical finding: no modality-level tag exists in Explorium

The autocomplete probe across `linkedin_category` returns broad industry buckets only. There is no Explorium category for "gene therapy", "AAV", "viral vector", or any specific modality. Gate 1 via Explorium can ONLY filter for the broad biotech/pharma/medical-device industry bucket. **Gate 2 (Exa web_fetch on company pipeline page) is required to enforce AAV-specificity.**

This is the design constraint that makes the two-stage gate non-negotiable for this play.

## How to use this file

When building or modifying a match-qualify-enrich workflow that targets the AAV gene therapy segment, the Gate 1 IF node should:

1. Pass-through if `linkedin_category` is in the IN-SCOPE category list AND `naics_code` is in the IN-SCOPE NAICS set.
2. Archive otherwise.

For a different sub-segment of biotech (e.g. cell therapy, mRNA, peptide), re-run the autocomplete probe with that segment's seed terms and create a separate taxonomy file. Do not edit this file to add other segments — it represents the AAV taxonomy specifically.

## Re-probing procedure

To refresh this file:

1. Run `mcp__explorium__autocomplete` with `field: linkedin_category` and the segment's seed keywords.
2. Run `mcp__explorium__autocomplete` with `field: naics_category` and "biotechnology" (or the segment's noun).
3. Cross-reference the returned values against the segment artifact's hard-filter exclusions.
4. Update the IN/OUT tables above. Date-stamp the refresh.
