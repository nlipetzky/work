## Q1_match_fetch_enrich_process

**Recommended n8n Workflow Patterns and Node Sequences**
Building effective go-to-market (GTM) agents in n8n requires structured data handling and specific node sequences. Explorium dictates that **every workflow must be a two-step process: you must first use the `Match` or `Fetch` endpoints to obtain unique Explorium IDs, and only then use those IDs in the `Enrich` or `Events` endpoints** [1-4].

*   **Error Handling:** A common error handling pattern is placing an `IF` node immediately after a `Match` node to check if the returned "Business ID" or "Prospect ID" is empty [5, 6]. This prevents the workflow from breaking if a domain doesn't return a valid match. Another pattern involves using an `IF` node and a code node to validate API calls generated by an AI agent; if the LLM hallucinates an invalid JSON schema, the `IF` node loops the failure back to the AI for refinement [7-10].
*   **Batching and Looping:** Because AI agents in n8n do not support auto-pagination, you must use a **`Loop Over Items` node with a batch size of 1** when feeding multiple enriched contacts or companies into an AI agent [11-13]. 
*   **Integration Patterns:** Explorium recommends using native n8n nodes or HTTP requests to pull initial triggers (e.g., Salesforce reports, form submissions, or Google Calendar events) [14-17]. It is also highly recommended to use **code nodes (JavaScript) to clean and structure merged JSON data** before feeding it to an AI agent, ensuring the LLM can easily digest the context [18-20].

**Recommended Architecture for a Match -> Qualify -> Enrich -> Outreach Pipeline**
Based on the Inbound, ABM Expansion, and Outbound templates, the ideal architecture flows as follows:
1.  **Match:** Trigger the workflow (e.g., via a new Salesforce lead or form submission) and extract the company name and domain. Feed this into the Explorium `Match` node to retrieve the Business ID [14, 21, 22].
2.  **Qualify:** Use the `Enrich` node to pull a lightweight bundle like firmographics and technographics, and optionally query internal usage data (e.g., Databricks or Mixpanel) [3, 23, 24]. Feed this context into an AI Agent node configured to score the lead. Route the output into an **`IF` node to check if the lead score is "high priority"** [4, 25].
3.  **Deep Enrich (Expansion):** For high-priority leads, pass the Business ID into the `Fetch Prospects` node to find relevant contacts (e.g., directors in the sales department) [26-28]. Then, run those prospect IDs through the `Enrich` node to pull professional profiles and valid contact information [29, 30].
4.  **Outreach:** Pass the enriched prospects through a `Loop Over Items` node into a two-agent setup. First, a **Research Agent** gathers context and pain points. Second, an **Email Writer Agent** uses that research to draft a personalized message [13, 30-33]. Finally, use a Gmail node to save the output as a draft or push the structured tasks into your CRM [34-36].

**Provided GTM Agent Templates**
Explorium shares five distinct n8n agent templates throughout their academy:
*   **Slack AI Agent:** A natural language bot residing in Slack that uses the Explorium MCP to answer data queries on demand [37-39].
*   **Inbound GTM Agent:** Automatically evaluates new leads, enriches them with company intelligence and product usage data, and qualifies them into structured Salesforce tasks [14, 17, 40].
*   **ABM Expansion Agent:** Extends the inbound agent by taking high-scoring accounts, finding additional decision-makers at those companies, and pushing them to the CRM [41, 42].
*   **Meeting Prep Agent:** A scheduled workflow that scans your calendar, filters for external attendees, conducts deep research on the companies and individuals, and sends a concise prep brief to Slack [15, 43].
*   **Outbound GTM Agent:** Translates natural language prompts (e.g., "Find fintech CMOs") into API calls, fetches and enriches those prospects, and automatically writes highly personalized cold outreach drafts [44, 45].

**Common Pitfalls and Anti-Patterns**
*   **The "Do-Everything" Agent:** Explorium strictly warns against building a single massive AI agent to handle an entire end-to-end workflow. This creates a "black box" that is difficult to debug or improve. **Instead, you should build multiple "narrow responsibility" agents** (e.g., separating a research agent from an email-drafting agent) [30, 31, 33, 46, 47].
*   **Treating Agents as Just Prompts:** Building an agent is not just about writing one large ChatGPT prompt. It is a full system that requires data ingestion, enrichment layers, routing logic, and error handling [37].
*   **Removing the Human-in-the-Loop for High-Risk Actions:** Agents should be treated as "co-pilots" rather than running on full autopilot for risky tasks. For outbound outreach, Explorium strongly advises configuring the final step to **create a Gmail draft rather than sending the email automatically**, allowing a human to review the AI's work [35, 36, 48-50].

## Q2_lead_qualification_SOPs

Building on our previous discussion about Explorium's automated workflows, Explorium teaches that lead qualification and go-to-market (GTM) motions should transition from rigid, manual processes to dynamic, AI-driven systems. By using autonomous agents, companies can process massive volumes of data and execute complex standard operating procedures (SOPs) with the speed and consistency that human teams cannot match [1-3]. 

Here is how Explorium recommends structuring these different workflows:

### Lead Qualification: Defining ICPs, Scoring, and Routing
Explorium recommends evaluating leads by continuously comparing inbound CRM data against an Ideal Customer Profile (ICP) using both internal usage metrics and external B2B data [4, 5]. 

*   **Segment Definition & Filtering:** The agent begins by using Explorium's "Match" and "Enrich" endpoints to pull deep data on an incoming company [6]. To see if a company fits the ICP, the agent filters based on strict parameters, such as analyzing if a company operates in a target industry (e.g., software or fintech), its revenue range, and whether its employee headcount signifies it is an SMB or an enterprise [6, 7].
*   **Scoring:** Instead of simple point-based lead scoring, Explorium recommends using an AI agent to conduct a comprehensive fit assessment. The agent evaluates the lead by analyzing **"how they're using the API to understand their use case, assesses company fit, and identifies key signals"** [8].
*   **Routing (Qualified vs. Disqualified):** Based on its analysis, the AI agent outputs a definitive score (such as "high priority" or "medium priority") and generates a structured output **"saying whether the lead is hot nurtured cold and something that your company should go after"** [9, 10]. From there, the workflow uses a routing logic node (such as an `IF` node in n8n); if a lead is "high priority," it is automatically routed to the sales team as a structured task in Salesforce, alongside recommended talking points [8, 11]. 

### Recommended Signals, Attributes, and Events
To properly enrich and score these leads, Explorium recommends using a diverse mix of data signals:
*   **Firmographics:** Standard metrics like employee range, revenue, and NAICS/SIC industry codes [12].
*   **Technographics:** The specific software or tools in a company's tech stack [13].
*   **Business Events:** Real-time triggers that indicate a lead is hot, such as a **"new product launch, new office opening, maybe some sort of new partnership"** or recent funding rounds and acquisitions [14, 15].
*   **Strategic Insights:** Broader market context, such as competitive landscape details, workforce hiring trends (e.g., expanding an engineering team), or risk factors pulled from 10-K filings [13, 16].
*   **Internal Product Usage:** For product-led models, tracking exactly which features or API endpoints a lead is testing during a free trial [5, 17]. 

### Account-Based Marketing (ABM) Framework
Explorium's ABM framework is designed to automatically capitalize on highly qualified accounts. They note that **"when a company scores well from the inbound agent we are then going to build an agent to expand on that account finding relevant leads at that same company"** [18]. 
*   Once a target account is verified, the agent uses the "Fetch Prospects" endpoint to find other individuals at that exact company [19]. 
*   It filters for specific decision-makers by searching for targeted job levels (e.g., directors, senior employees) in relevant departments (e.g., sales, strategy) [20]. 
*   The agent then enriches these new prospects to get their professional emails and job titles, and automatically pushes this new list of contacts into the CRM under the target account [21-23].

### Meeting Preparation Framework
To prepare sales reps for calls, Explorium recommends a scheduled, calendar-driven agent. They describe this as **"a researchheavy agent that prepares sales reps for meetings The agent is going to combine multiple data sources and produce structured research outputs that are actionable by these sales reps"** [24].
*   The workflow automatically scans a calendar (like Google Calendar) every morning and filters out internal meetings [25, 26].
*   It splits the research into two simultaneous tracks: one loop pulls the company domain to research firmographics and recent business events, while the other loop pulls the attendees' emails to research their contact info, career history, and recent LinkedIn posts [27-34].
*   An AI agent synthesizes this data into a comprehensive research brief containing conversation starters, pain points, and fit reasoning, and routes it to the rep via Slack [35, 36].

### Large-Scale Outbound Framework
For cold outbound, Explorium utilizes a multi-step orchestration that **"takes you from a simple natural language query all the way to personalized researchbacked email drafts"** [37].
*   **Natural Language to API:** A user types a conversational prompt (e.g., "Find me five marketing leaders at fintech startups who joined in the past year"). An AI agent dynamically translates this text into a structured, validated API search to pull the precise prospects requested [37-40].
*   **Task Splitting:** Explorium specifically recommends splitting the AI workload into "narrow responsibility agents" for better results [41, 42]. They route the prospect data first to a **"Research Agent"** that compiles context about the prospect's pain points and tech stack. That data is then handed off to an **"Email Writer Agent"** that drafts the customized copy [42-47]. 
*   **Human-in-the-Loop:** To ensure quality control at scale, the framework pushes the final outreach copy into a platform like Gmail or Outreach specifically as *drafts*, allowing a human rep to manually review and approve the emails before they are actually sent [48-50].

## Q6_endpoints_full_list

**Match (Business & Prospect)**
*   **Purpose:** The required first step when you already know a company or person. It returns a unique Explorium ID [1-5].
*   **Required Inputs:** For businesses, a company name and/or domain [3, 5, 6]. For prospects, an email, LinkedIn URL, name, or business ID [7-10].
*   **Returned Fields:** A unique Explorium Business ID or Prospect ID [3, 6, 8, 9, 11].
*   **Rate Limits:** Detailed in the documentation, though exact numbers are not specified in the course materials [7, 12]. 
*   **Cost Considerations:** This is a completely free process that uses zero credits [3, 13].
*   **When to Use vs. Skip:** **Always use this first** when you have a company/prospect name, domain, or email [1, 2, 4]. Skip if you are trying to discover *new* prospects or businesses based on general filters (use Fetch instead) [4].

**Fetch (Businesses & Prospects)**
*   **Purpose:** Discovers multiple new IDs for a general category of businesses or prospects based on your Ideal Customer Profile (ICP) [4, 7, 11, 12, 14, 15].
*   **Required Inputs:** Specific filters like country/state, employee count, revenue range, industry, NAICS/SIC, job title/level, department, and contact info availability (e.g., valid email or phone) [15-22]. 
*   **Returned Fields:** A paginated list of matching records including their unique Business or Prospect IDs, names, domains, basic firmographics, or hashed emails [15, 18-20, 23-25].
*   **Rate Limits:** Detailed in the documentation [7, 12].
*   **Cost Considerations:** Uses credits to generate lists of net-new records. 
*   **When to Use vs. Skip:** Use to build targeted lists of new accounts or individuals matching certain criteria [11, 14, 15]. Skip if you already know the specific domains or names you want to target (use Match instead) [4].

**Enrich (Business & Prospect)**
*   **Purpose:** Pulls deep, detailed data and insights using an Explorium ID [1, 2, 4, 26].
*   **Required Inputs:** An Explorium Business ID or Prospect ID [2, 27, 28].
*   **Returned Fields (Business):** Firmographics (description, industry, NAICS, SIC, employees, revenue), Technographics, Competitive Landscape (differentiation), Strategic Insights, Company Ratings, Funding, Workforce trends, and Acquisitions [27, 29-32].
*   **Returned Fields (Prospect):** Contact information (professional emails, phone numbers), LinkedIn posts, and Professional profiles (experience, skills, age, education) [28, 33-37].
*   **Rate Limits:** Detailed in the documentation [7, 12].
*   **Cost Considerations:** This pulls expensive data and uses credits. 
*   **When to Use vs. Skip:** Use *after* matching or fetching when you need deep intelligence for personalization, qualification, or outreach [1, 2, 34]. Skip during early pre-qualification stages if you only need to filter out bad-fit companies to save credits.

**Statistics (Business)**
*   **Purpose:** Provides aggregated metrics on your Total Addressable Market (TAM) or target ICP [1, 11, 14, 16].
*   **Required Inputs:** General filters like state, employee count, or industry category [16].
*   **Returned Fields:** Aggregated statistics showing the distribution of businesses by location, company size, and revenue range [16].
*   **Rate Limits:** Detailed in the documentation [7, 12].
*   **Cost Considerations:** Uses credits.
*   **When to Use vs. Skip:** Use to refine your criteria and understand market size *before* pulling expensive company or prospect lists [16, 38]. Skip if you already have highly targeted, pre-defined parameters.

**Events (Business & Prospect)**
*   **Purpose:** Monitors and retrieves recent activities or real-time triggers for companies and people [33, 39-41].
*   **Required Inputs:** For businesses: Business ID, event types (e.g., new product launches, new investments, funding rounds, strategic partnerships, office openings), and a date range (defaults to the last 3 months) [41-44]. For prospects: Prospect ID, and event types (e.g., role changes, company changes, new positions) [44, 45].
*   **Returned Fields:** Event descriptions, announcement links, titles, and snippets [42, 45].
*   **Rate Limits:** Detailed in the documentation [7]. Can be configured via push webhooks for real-time monitoring [39, 44, 46].
*   **Cost Considerations:** Uses credits.
*   **When to Use vs. Skip:** Use when prioritizing hot leads or finding timely, relevant triggers for cold outreach [40, 47]. Skip if you only need static, historical profile data.

**Autocomplete Endpoint**
*   **Purpose:** Helps users find the exact syntax or acceptable parameters in the Explorium database [39, 46].
*   **Required Inputs:** An incomplete phrase or similar industry string [46].
*   **Returned Fields:** The closest valid parameter or syntax accepted by Explorium [39, 46].
*   **Rate Limits:** Detailed in the documentation [7].
*   **Cost Considerations:** A lightweight helper tool.
*   **When to Use vs. Skip:** Use when you do not know the exact acceptable filter syntax (like specific NAICS codes or industry strings) [39, 46]. Skip if you already know the exact parameter values.

**Explorium MCP (Model Context Protocol) Server/Tool**
*   **Purpose:** Connects LLMs (like Claude) directly to Explorium's data, allowing an AI agent to read API documentation, translate natural language into structured API calls, and perform supplemental research [31, 44, 48-51].
*   **Required Inputs:** Natural language queries, LLM context, or events [44, 51-53].
*   **Returned Fields:** Tailored API calls, enriched summaries, formatted tables, or customized cold emails [21, 44, 53, 54].
*   **Rate Limits / Cost Considerations:** Inherits the limits and costs of the underlying API endpoints it autonomously calls. 
*   **When to Use vs. Skip:** Use within an AI agent node (in n8n or an MCP chat app) when you want the AI to dynamically research leads, evaluate qualification fit, or translate natural language chat into API filters [31, 44, 48, 53]. Skip in rigid, predefined automation workflows that only require hard-coded API nodes [55, 56].

## Q4 Personalized outreach

Explorium teaches that the most effective way to generate personalized outreach is to avoid relying on a single "do-everything" AI agent. Instead, they recommend splitting message generation into a multi-agent workflow with **narrow responsibility agents**.

Outbound message generation uses a two-agent pattern:
1. **Research Agent**: takes enriched data + Explorium MCP, focuses on prospect activity, tech stack, pain points.
2. **Email Writer Agent**: takes research agent output, drafts cold outreach referencing role, company context, soft CTA.

**Personalization data points:**
- Company: firmographics (size/industry/revenue), technographics (e.g., Salesforce, Snowflake), competitive landscape, business events (product launches, office openings, funding).
- Prospect: job history, level/department, skills, recent LinkedIn activity.
- Product usage (inbound): query Databricks/Mixpanel to see which features the lead is testing.

**Human-in-the-loop philosophy:**
Output drafts to Gmail / Outlook / Lemlist — never auto-send. Allow rep to manually review, tailor, verify before sending.

**Not covered in sources:** cadence structure, multi-step sequencing, deliverability, suppression lists, reply routing. Explorium said cadence setup is covered in their live hackathons but not the course material.

## Q5 Data quality / QA / human-in-the-loop

- Explorium aggregates 50+ data sources (Bombora, Zero Bounce, Builtwith, Ocean.io) into a single harmonized API — claims this solves entity-matching / messiness vs. web search.
- Emphasizes **timely + trusted intelligence**: data refreshed on a known cadence so agents act on fresh signals.
- **Missing match handling**: IF node after Match checks Business ID / Prospect ID is not empty; skip the loop if empty to avoid wasted credits.
- **Validation node**: after AI-generated API calls, validate filter values against allowed schema; loop bad calls back to the AI to retry.
- **Human-in-the-loop**:
  - Output drafts, not auto-sends.
  - Treat agent as co-pilot first, graduate toward autopilot as trust builds.
  - Reserve guardrails for high-risk actions.

**Not covered:** parent-vs-subsidiary handling, dedup methods, source attribution mechanics.
