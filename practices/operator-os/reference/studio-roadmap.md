# Studio Roadmap — all the work, in proper order

Hand-built by Atlas, 2026-06-25, from the live evidenced state of every system (System State) +
the goals/vision + the dependency structure. This is deliberately manual: sequencing 37 systems is a
judgment call to prove by hand before it becomes a system (Compass Planning). The **method** below is
the part that gets systematized; the **sequence** is its output for today's state.

## The method (how the order is derived)

Three lenses, applied in priority — not a flat score:

1. **Binding constraint first.** What single thing, unsolved, blocks everything? Right now: **revenue
   — leads to Will / CIPO** (G1, the engine's first proof). Money buys time and proves the asset
   thesis. Nick has said directly: this is THE thing; everything else is secondary.
2. **Foundation-first within a goal.** Build the system whose OUTPUT others consume before the
   consumers. You cannot run what has no inputs. (CIPO: the context that produces offer/copy/TAM
   /targeting comes before the website or the send.)
3. **Leverage / compounding** for everything off the critical path (Naval): prefer reusable
   code/media assets over one-off work.

**Gating reality (from System State): 0 systems are operating, 0 beta.** Almost everything is a
BUILD. So this roadmap is mostly a build sequence — `build` what's stub/emerging, `iterate` what's
building, `run` only beta+/operating (nothing yet). Re-derive the order whenever the evidenced state
or the binding constraint changes.

THE priority. Foundation-first within it — the dependency chain, in order:

1. **Expert-Liaison + Demand-Context** *(build)* — get Will's signal, synthesize the **offer · ICP ·
   copy inputs · TAM criteria · targeting**. The foundation; everything downstream needs its output.
   (expert-liaison is the furthest along — assets + a wired trigger; demand-context is emerging.)
2. **Voice-Authoring** *(build)* → the offer + cold copy, from the context.
   **RevOps-Engine + Signal-Targeting** *(iterate/build — revops-engine is the most-built system, 18
   assets)* → turn the TAM criteria into the actual **cohort (companies + contacts)**.

4. **Voice-Listening** *(build)* → capture replies, book Will's calendar. Closes the loop to revenue.
- **Done when:** CIPO outreach is live — evidenced-qualified contacts, real copy, replies and meetings
  landing on Will's calendar. First leads delivered.
- (Note: the "CIPO website" is a downstream output of Voice-Authoring/Forge, not a foundation — it
  comes *after* the offer exists, not first.)

## Why this order, in one line

Revenue is the constraint → build the CIPO context first (it feeds everything) → then copy + cohort →
then send → then close/bill → then compound with audience and keep-live infra. Everything that doesn't
move a lead toward Will, or compound an owned asset, waits.
